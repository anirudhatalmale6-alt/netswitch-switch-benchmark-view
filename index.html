Live meter, refreshing every second. Device RAM and CPU cores are the phone's real hardware; CPU load is measured from how late the main thread ticks (event-loop lag), and the Chrome heap is this tab's RAM. The Android "Process system isn't responding" popup fires when either one maxes out: heap crosses the per-app RAM limit (a big cloud file pulled whole into HTML, or too many tabs), or the CPU is pinned so the main thread can't answer in time. Fix both ways: close pages before the jam and load big files in chunks, not all at once. The phone camera stalls for the same reason — it needs a large free RAM buffer per frame.
