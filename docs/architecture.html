Per-device DB (encrypted)

One database file per hardware device: data/devices/<id>.db, encrypted at rest with AES-256-GCM (Node crypto). Last 500 snapshots per device. Transport secured by TLS.
Data flow — field device measures itself live → optionally POSTs a Health Snapshot → server stores it in that device's encrypted DB → maintenance staff read /api/device/history & /api/health to see which device or link is struggling.   Line length = server↔phone ping (e.g. ping bbc.co.uk to the phone), feeding the MPEG-4 / broadband delivery estimate to 30,000 post addresses, compared across fixed-line / mobile-Wi-Fi / 5G.   Honesty rule: everything measurable is measured; anything impossible from an app (raw traceroute, radio control, locating a phone number) is either done server-side for real or labelled as a model — never faked.
