6GGW — 5G Gateway Router

Sources & Test Guide  ·  Client v2.49.0  ·  Server v0.4.0  ·  for testers

The product ships in two halves. The client is an installable app (PWA) that runs on any phone, tablet or computer with a browser — nothing to install from a store. The server is a small zero-dependency Node program that runs on a real box (HCL / Fujitsu / a laptop) and does the measurements a browser physically can't. One server can also serve the client, so a single box ships both halves.

Test the CLIENT today (no server needed)

  1. Unzip the client and open index.html in Chrome (phone or desktop). On a phone you can tap Add to Home screen to install it as an app.
  2. Walk the tabs. The live ones to check first:
    • CPU·GPU — RAM / CPU / Chrome-heap live meter (the "Process system isn't responding" jam guard).
    • NET DATA — Stream Delivery Estimate: pick a service (Spotify / Tidal / BBC), set % online at peak and split-for-cache, and read the origin vs edge node counts.
    • 6G GATEWAY — route / line-length view.
  3. In CPU·GPU → HEALTH SNAPSHOT, each tester can capture their device's real stats and (tomorrow, once the server is up) send them to it.
Three testers: each just opens the same client on their own device — phone, tablet, laptop. No accounts, no install. Their device stats differ, so it's a good spread.

Run the SERVER tomorrow (rc build)

On the test box, unzip the server folder and run it as the release candidate. Give each location its own device id.

cd server
CHANNEL=rc DB_KEY="pick-a-secret" node server.js        # listens on :8090
# open http://<server-ip>:8090/  — it serves the client too

In the client (CPU·GPU tab) set Maintenance server URL to http://<server-ip>:8090 and a device id (e.g. site-A, site-B per location), then SEND SNAPSHOT TO SERVER. Each device's snapshots are stored in its own AES-256 encrypted DB on the server.

Security watch & firewall check

The moment the server starts, a round-the-clock watch runs on a two-hand clock: a fast tick samples request/error rate and the listener, a slow tick rolls up and runs a real firewall / egress self-test. Everything is logged to data/security/security-<date>.log.

curl http://<server-ip>:8090/api/security          # fw_ok, counters, recent alerts
curl "http://<server-ip>:8090/api/security/log?n=50"

Or read it in the app: CPU·GPU → SECURITY WATCH (SERVER).

Back up the network data (tape / LTO / offsite)

node backup.js                                     # -> data/backups/backup-rc-<ts>.6ggwbak.gz (+ .sha256)
node restore.js data/backups/backup-....6ggwbak.gz --verify   # verify an archive

Blobs stay encrypted inside the archive, so it's safe to copy straight onto tape. After the test run, node backup.js gives you one file to send back.
